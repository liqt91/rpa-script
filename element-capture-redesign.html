元素捕获侧板 UI 重设计方案（修订版）

1. 现状与问题

当前 extension/sidepanel.html 采用垂直堆叠，存在以下体验问题：

  • 信息密度过高：选择器预览、校验结果、目标模式、循环锚点全部挤在底部固定栏，编辑锚点时空间不足。
  • 截图占用首屏：截图区域固定 140px，捕获时抢眼但并不需要一直展开。
  • 目标模式与 Tab 重复：底部有“目标模式 [单个/列表 ▼]”，顶部又有 XPath-单个 / XPath-列表 / CSS-单个 / CSS-列表 四个 Tab，二者互相影响但位置分离，容易选错。
  • 锚点不可见/难操作：anchorBox 只在元素已被锚定时才出现，且仅展示两个输入框，无法直观看到锚点元素，也没有重新捕获入口。
  • Drission 选择器占用 Tab：对普通用户而言 Drission 是内部实现细节，却与 XPath/CSS 并列成一级 Tab，增加认知负担。

2. 设计目标

  1. 布局合理：按“命名/看图 → 选 family → 选模式 → 选方案 → 调锚点 → 保存”的顺序组织。
  2. 截图可折叠：首屏只显示截图入口按钮，点击后展开大图，不占用编辑空间。
  3. 目标模式与 XPath/CSS 关系直觉化：用“Family Tab + 单个/列表切换”替代 4 个平铺 Tab，目标模式不再单独下拉。
  4. 锚点可查看、可重新捕获：把循环锚点提升为独立卡片，展示锚点元素摘要并提供重捕获入口。
  5. 不展示 Drission 选择器：侧板界面完全隐藏 Drission；Drission 选择器仍由后端保存供 Python 导出使用。
  6. 向后兼容：relativeSelector / anchorSelector / anchorMode 数据模型不变。

4. 模块详细设计

4.1 头部（固定）

保留“环境”下拉和“流程”下拉，连接状态点放到流程选择框左侧：

环境 生产 ▼ ● 流程 选择流程... ▼

4.2 元素名称与截图按钮

元素名称 [输入元素名称          ] [📷]
  • 截图不再默认展开。
  • 名称输入框右侧放置一个小按钮（图标或缩略图），hover 提示“查看截图”。
  • 点击后，在名称行下方展开一个截图面板；再次点击或点击关闭按钮收起。
  • 无截图时按钮置灰，提示“暂无截图”。

4.3 选择器类型 + 匹配范围

优化核心：把“选择器 family”和“匹配范围”拆成两个互相独立的控件。

控件作用默认值
Family Tab切换 xpath / css上次保存或 xpath
单个/列表切换切换 targetModesingle

交互规则：

  • 切换 Family Tab：候选列表按 family 过滤；当前选择器前缀自动切换（xpath: ↔ css:）。
  • 切换 单个/列表：候选列表按 isList 过滤；targetMode 同步更新。
  • 不再显示“目标模式”下拉框，避免与 Tab 重复。
  • 保存时，根据当前 Family + 单个/列表状态生成：
    • selectorFamily
    • targetMode
    • selector（当前选择器文本，已带 family 前缀）

这样用户只需要“选 family → 选范围 → 点候选方案”三步，逻辑与界面一一对应。

4.4 推荐方案与当前选择器

  • 推荐方案列表按当前 family + 模式过滤。
  • 选中方案后回填“当前选择器”文本域，并触发页面高亮。
  • 文本域允许用户手动微调；校验按钮对文本域内容执行页面匹配。
  • 校验结果展示匹配数、是否唯一、score。

5. 重新捕获锚点流程

方式 A：一键重算（一期）

  1. 用户点击“重新捕获锚点”。
  2. sidepanel.js 发送 recomputeAnchor 消息到 content.js，携带当前目标元素信息。
  3. content.js 重新执行 detectListFamily + computeRelativeSelector。
  4. 返回新的 anchorSelector / relativeSelector，anchorMode='auto'。
  5. 侧板刷新卡片。

方式 B：手动指定锚点元素（二期）

  1. 用户点击“手动指定锚点”。
  2. content.js 进入高亮模式，高亮所有候选列表项。
  3. 用户点击某个列表项后，以该项为新锚点重新计算 relativeSelector。
  4. 返回侧板，anchorMode='manual'。

6. 需要移除/隐藏的内容

  • sidepanel.html 中的 <button class="tab" data-tab="drission">Drission</button>。
  • sidepanel.js 中 Drission 候选过滤、Drission 标签渲染。
  • 原“目标模式”下拉框；目标模式由 单个/列表 切换隐式决定。
  • 固定的截图区域；改为可折叠按钮。
