1. CSS：严格限制在原生范围

核心原则：CSS 选择器必须能被 document.querySelectorAll 原生解析。
  • 禁止生成 :contains("text")、/deep/、::shadow 等非标准语法。
  • innerText 在 CSS 标签下可勾选，但不输出到选择器。
  • 校验直接走 querySelectorAll，不再做 jQuery shim。

2.2 位置序号 [n] 惩罚

每个 :nth-of-type() 或 XPath [n] 扣 50 分。位置信息是最后手段。

3. 校验层

单一入口 verifyLocator(syntax, family)：

  • xpath：去掉 xpath: 前缀后直接 document.evaluate。
  • drission：verse: / text= / tag:*@text() 转给 resolveAllForVerify。
  • css：去掉 css: 前缀后 querySelectorAll。

4. 双向一致性

4.1 推荐 → UI

候选附带 pathMapping，标明每个 segment 对应 path[i] 的哪个层级：

css:body > div > nav > a
xpath://body/div/nav/a[3]
mapping: [0, 1, 2, 3]

侧板直接按映射勾选，不再反向猜测。

4.2 UI → 选择器

同一组 checkbox 状态，CSS/XPath 语义一致：

5. 候选截取：各家族 Top 10

不再使用全局 Top 10，改为 CSS / XPath / Drission 各取前 10，避免 XPath 被 CSS 大量挤出侧板。 
function pickCandidatesByFamily(all, limitPerFamily = 10) {
  const byFamily = {};
  for (const c of all) {
    const family = c.family || c.type || 'css';
    (byFamily[family] ||= []).push(c);
  }
  return Object.values(byFamily).flatMap(arr => arr.slice(0, limitPerFamily));
}

6. 动态属性黑名单

新增 isFragileAttr(name)，采集阶段直接过滤：

  • style
  • onclick、onchange 等事件处理器
  • data-react*、data-v-*
  • _ngcontent-*、_nghost-*
  • 动态 ARIA：aria-owns、aria-activedescendant、aria-busy、aria-live、aria-relevant
